# Discord welcome — channel structure

Suggested channel layout for the Ferrox Discord server (or any chat server hosting the community):

| Channel | Purpose |
|---|---|
| `#announcements` | Releases, security advisories, governance changes. Read-only for non-maintainers. |
| `#help` | "How do I…?" questions. Free-form. Threads encouraged for any issue lasting more than a few messages. |
| `#development` | Discussion about open PRs, design decisions, ADR proposals. |
| `#show-and-tell` | Cool things people are building with Ferrox. Screenshots, blog posts, benchmarks. |
| `#triage` | Optional, maintainer-only — for triaging issues without spamming `#development`. |

## Welcome message template

```
👋 Welcome to **Ferrox** — AWS S3-compatible object storage in Rust.

📚 Docs: https://ferrox-rs.github.io/ferrox
💻 Repo: https://github.com/ferrox-rs/ferrox
🔒 Security policy: https://github.com/ferrox-rs/ferrox/blob/main/SECURITY.md
📜 Code of conduct: https://github.com/ferrox-rs/ferrox/blob/main/CODE_OF_CONDUCT.md

A few channel pointers:
- New here, or troubleshooting? → #help
- Working on a PR? → #development
- Built something? → #show-and-tell

Vulnerabilities go through GitHub Security Advisories, not here — see SECURITY.md.
Be excellent to one another. 🦀
```
